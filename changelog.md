## 1.0.0 - 2026-04-07
* Several public SDK classes have been removed in this release. The following types no longer exist and any references to them must be updated:
* `UpdateAssistantDtoVoicemailDetection` and `AssistantOverridesVoicemailDetection` (voicemail detection union types)
* `CallControllerFindAllPaginatedRequest` and `CallControllerFindAllPaginatedRequestSortOrder` (call listing request/enum)
* `SessionsListRequest` (session listing request)
* `RetryMiddleware` (internal HTTP retry middleware)
* Please update your code to use any replacement types provided by the new API surface.
* Several public types have been removed in this release as part of an API model consolidation:
* `AssistantVoicemailDetection`, `CreateAssistantDtoVoicemailDetection`, `CreateWorkflowDtoVoicemailDetection`, and `UpdateWorkflowDtoVoicemailDetection` have been deleted. Consumers should use the concrete voicemail detection plan types (`GoogleVoicemailDetectionPlan`, `OpenAiVoicemailDetectionPlan`, `TwilioVoicemailDetectionPlan`, `VapiVoicemailDetectionPlan`) directly.
* `UpdateSupabaseCredentialDto` has been removed.
* The `speakerLabel` property has been removed from `BotMessage`. Any code reading `$botMessage->speakerLabel` must be updated.
* Several breaking changes have been made to the SDK in this release.
* The `WorkflowVoicemailDetection` and `WorkflowUserEditableVoicemailDetection` classes have been removed from `Vapi\Types`. Callers should use the appropriate provider-specific voicemail detection plan classes (`GoogleVoicemailDetectionPlan`, `OpenAiVoicemailDetectionPlan`, `TwilioVoicemailDetectionPlan`, or `VapiVoicemailDetectionPlan`) directly.
* `AssistantsListRequest` has been renamed to `ListAssistantsRequest`; update all import and instantiation sites accordingly.
* All `AssistantsClient` methods (`list`, `create`, `get`, `delete`, `update`) and `AnalyticsClient::get()` now return nullable types (e.g. `?Assistant`, `?array<Assistant>`); callers must guard against `null` return values.
* **Breaking changes in this release:**
* `CallsListRequest` has been renamed to `ListCallsRequest`. Update all references accordingly.
* The `callControllerFindAllPaginated()` method has been removed from `CallsClient` with no direct replacement.
* Return types for `CallsClient` methods (`list`, `create`, `get`, `delete`, `update`) and all `CampaignsClient` methods are now nullable — callers must handle `null` responses.
* The `voicemailDetection` field on `UpdateAssistantDto` now accepts a union of provider-specific plan types (`GoogleVoicemailDetectionPlan`, `OpenAiVoicemailDetectionPlan`, `TwilioVoicemailDetectionPlan`, `VapiVoicemailDetectionPlan`, or the string enum `UpdateAssistantDtoVoicemailDetectionZero`) instead of the previous `UpdateAssistantDtoVoicemailDetection` type.
**New features:**
* `SessionCreatedHook` is now supported in the `hooks` array on assistant DTOs.
* DateTime filter parameters in list requests are now correctly serialized.
* The SDK now supports `squadId` and `dialPlan` as new optional fields on campaign create and update requests, enabling calls to be routed through squads or across multiple phone numbers. The `phoneNumberId` and `customers` fields on `CreateCampaignDto` are now optional (required only when `dialPlan` is not provided).
* The `ChatsListRequest` class (formerly `ChatsListRequest`) has been renamed to `ListChatsRequest`, and its `workflowId` filter has been removed; new filters `id`, `assistantIdAny`, and `previousChatId` have been added. All `ChatsClient` methods now return nullable types to handle empty API responses. The underlying HTTP client has been migrated from Guzzle to PSR-18, so any project using this SDK must provide a PSR-18-compatible HTTP client.
* All public methods on `EvalClient` and `FilesClient` now return nullable types (e.g., `?Eval_`, `?File`, `?array<File>`). Callers should update their code to handle `null` return values, which are returned when the server responds with an empty body. DateTime-based query filter parameters are now correctly serialized before being sent to the API.
* Several public classes in the `PhoneNumbers` namespace have been renamed: `PhoneNumbersListRequest` → `ListPhoneNumbersRequest`, `PhoneNumbersListResponseItem` → `ListPhoneNumbersResponseItem`, `PhoneNumbersCreateRequest` → `CreatePhoneNumbersRequest`, `PhoneNumbersCreateResponse` → `CreatePhoneNumbersResponse`, `PhoneNumbersGetResponse` → `GetPhoneNumbersResponse`, `PhoneNumbersDeleteResponse` → `DeletePhoneNumbersResponse`, `PhoneNumbersUpdateRequest` → `UpdatePhoneNumbersRequest`, and `PhoneNumbersUpdateResponse` → `UpdatePhoneNumbersResponse`. All `PhoneNumbersClient` methods (`list`, `create`, `get`, `delete`, `update`, `phoneNumberControllerFindAllPaginated`) now return nullable types — callers must handle a `null` return value for empty responses. Update any type hints, `instanceof` checks, or direct class references in your code to use the new class names.
* Several public classes have been renamed for naming consistency: `PhoneNumbersUpdateRequest` is now `UpdatePhoneNumbersRequestBody`, `PhoneNumbersDeleteResponse` is now `UpdatePhoneNumbersResponse`, `SquadsListRequest` is now `ListSquadsRequest`, and `SessionsListRequest` is now `ListSessionsRequest`. All methods on `ProviderResourcesClient` and `SessionsClient` now return nullable types (e.g. `?ProviderResource`, `?Session`, `?SessionPaginatedResponse`) to correctly handle empty API responses — callers should add null checks where needed. `ListSessionsRequest` also gains new filter fields: `id`, `assistantIdAny`, `numberE164CheckEnabled`, `extension`, `assistantOverrides`, `number`, `sipUri`, `email`, `externalId`, `customerNumberAny`, `phoneNumberId`, and `phoneNumberIdAny`.
* The `StructuredOutputsClient` now exposes a new `structuredOutputControllerRun()` method for on-demand structured output extraction. The `UpdateStructuredOutputDto` gains three new optional fields — `type`, `regex`, and `compliancePlan` — and the model union now includes `anthropic-bedrock` as a supported provider. Note: `ToolsListRequest` has been renamed to `ListToolsRequest` and `SquadsListRequest` has been renamed to `ListSquadsRequest`; update any direct references to these classes. All CRUD methods on `SquadsClient` and `StructuredOutputsClient` now return nullable types to handle empty API responses.
* Several Tools API types have been renamed for consistency. Replace `ToolsListRequest` with `ListToolsRequest`, `ToolsListResponseItem` with `ListToolsResponseItem`, `ToolsCreateRequest` with `CreateToolsRequest`, `ToolsCreateResponse` with `CreateToolsResponse`, `ToolsGetResponse` with `GetToolsResponse`, `ToolsDeleteResponse` with `DeleteToolsResponse`, `ToolsUpdateRequest` with `UpdateToolsRequest`, and `ToolsUpdateResponse` with `UpdateToolsResponse`. All `ToolsClient` methods (`list`, `create`, `get`, `delete`, `update`) now return nullable types to handle empty response bodies. Two new tool variants — `sipRequest` and `voicemail` — are now supported in `CreateToolsRequest`.
* The `ToolsCreateResponse` class has been renamed to `CreateToolsResponse`. Consumers must update all references, `use` statements, and type hints from `ToolsCreateResponse` to `CreateToolsResponse`. Additionally, the SDK now supports three new tool types in the tools create response: `CodeTool`, `SipRequestTool`, and `VoicemailTool`.
* The `ToolsDeleteResponse` class has been renamed to `DeleteToolsResponse` (in the `Vapi\Tools\Types` namespace). Callers must update all type references and instantiation calls accordingly. Additionally, the delete-tool response union now supports three new tool variants: `CodeTool`, `SipRequestTool`, and `VoicemailTool`.
* The `ToolsUpdateResponse` class has been renamed to `GetToolsResponse`. Update any type hints, `new` expressions, and `instanceof` checks accordingly; the file has moved to `src/Tools/Types/GetToolsResponse.php`. Three new tool variants are now supported in `GetToolsResponse`: `CodeTool`, `SipRequestTool`, and `VoicemailTool`.
* The `ToolsListResponseItem` class has been renamed to `ListToolsResponseItem`, and `ToolsUpdateRequest` has been renamed to `UpdateToolsRequestBody`. Update all references to these classes in your code to use the new names. Additionally, three new tool types are now supported: `CodeTool`, `SipRequestTool`, and `VoicemailTool`, each with corresponding factory methods and typed accessors on the union types.
* The `ToolsGetResponse` class has been renamed to `UpdateToolsResponse`. Existing code referencing `ToolsGetResponse` must be updated to use `UpdateToolsResponse` instead.
* Additionally, the `voicemailDetection` property on `Assistant` and `AssistantOverrides` now accepts a union of provider-specific plan types (`GoogleVoicemailDetectionPlan`, `OpenAiVoicemailDetectionPlan`, `TwilioVoicemailDetectionPlan`, `VapiVoicemailDetectionPlan`) instead of the previous `AssistantVoicemailDetection` / `AssistantOverridesVoicemailDetection` types — existing code assigning or reading this property will need to be updated. New capabilities include support for `CodeTool`, `SipRequestTool`, and `VoicemailTool` in `UpdateToolsResponse`, new `vadAssistedEndpointingEnabled` and `speechModel` fields on `AssemblyAiTranscriber`, and a new `SessionCreatedHook` option in the `hooks` array.
* Several breaking changes are included in this release:
* The `Eval` class has been renamed to `Eval_` (file renamed from `Eval.php` to `Eval_.php`). Update all references from `Eval` to `Eval_`.
* `FallbackSpeechmaticsTranscriber` removes the `maxSpeakers`, `enablePartials`, `enableEntities`, `enablePunctuation`, and `enableCapitalization` fields. Remove any usage of these fields from your code.
* `EvalAnthropicModel` and `EvalGroqModel` now require a `messages` field in their constructor arrays.
* `EvalRun` now requires `cost` and `costs` fields.
* New capabilities have also been added: VAD configuration fields on ElevenLabs transcribers, profanity filtering for Deepgram, temperature/speaking rate controls for Inworld voices, subtitle type control for Minimax voices, encryption plan support for custom credentials, and expanded hook/voicemail detection union types.
* Several breaking changes have been introduced in this release:
* `GoogleVoicemailDetectionPlan`, `OpenAiVoicemailDetectionPlan`, and `HandoffDestinationAssistant` constructors now require a non-empty `$values` array (previously defaulted to `[]`), and each requires a new mandatory field (`provider` or `type`). Existing instantiations without these fields will fail at runtime.
* `GetChatPaginatedDto` has had the `workflowId` field removed; callers using this field should migrate to the new `assistantIdAny` or other available filters.
* `ProviderResource::$resource` has changed type from `ElevenLabsPronunciationDictionary` to `array<string, mixed>`; code accessing typed properties of this field must be updated.
* `ServerMessageResponseHandoffDestinationRequest::$destination` has changed type from `HandoffDestinationAssistant` to `array<string, mixed>`.
* `JsonSchema::$items` is now typed as `?JsonSchema` (previously `?array<string, mixed>`) and `$properties` is now `?array<string, JsonSchema>`.
* New capabilities include: optional `temperature` and `speakingRate` on `InworldVoice`, optional `promptName`/`promptVersion`/`traceName` on `LangfuseObservabilityPlan`, optional `assistant` (transient) and `sessionExpirationSeconds` on `OpenAiWebChatRequest`, a new `ServerMessageAssistantSpeech` server message type, and `turnId` tracking on model output and user-interrupted messages.
